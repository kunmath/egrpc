# egrpc

egrpc is a clean-room, client-only gRPC C++ library for embedded Linux (Yocto). It is
wire-compatible with unmodified upstream gRPC servers: code generated by upstream `protoc`
and `grpc_cpp_plugin` compiles and runs against egrpc unchanged. Its only runtime
dependencies are nghttp2, OpenSSL, and protobuf-lite — no Abseil, no c-ares, no vendored
transport.

The v0.1 scope covers unary calls with deadlines, server-streaming `ClientReader`, TLS with
ALPN `h2`, keepalive PING, reconnect with backoff and jitter, clean shutdown, and
`egrpc::resilient_stream`. Permanently out of scope: the server side, client/bidi streaming,
xDS, interceptors, compression, and async completion queues. Async, client-streaming, and
bidi symbols are compile-present but return `UNIMPLEMENTED` at runtime.

**Status: pre-alpha, milestone M0 (scaffolding).** Nothing here is functional yet; the tree
is CMake, CI, and API scaffolding only.

## Build quickstart

```
cmake -S . -B build && cmake --build build -j && ctest --test-dir build
tools/pin-codegen.sh        # installs pinned protoc + grpc_cpp_plugin into tools/bin/
```

Build dependencies (Debian/Ubuntu): `libnghttp2-dev libssl-dev libprotobuf-dev pkg-config`.

## Pinned codegen policy

Because `grpc::internal` is unstable upstream, one code-generator toolchain is pinned per
egrpc release and verified in CI. v0.1 pins the **gRPC v1.82.0** `grpc_cpp_plugin` (built
from SHA-256-verified source) and **protoc 35.0** (official prebuilt binary, SHA-256-verified).
`tools/pin-codegen.sh` fetches and builds both into `tools/bin/`. See
[docs/codegen-pinning.md](docs/codegen-pinning.md) for versions, checksums, and rationale.
